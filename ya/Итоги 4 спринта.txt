Базовая IDE

    Integrated Development Environment (IDE) — комплексный набор инструментов разработчиков и разработчиц для написания и отладки кода.
    Удалённые репозитории — это модификации проекта, которые хранятся в интернете или ещё где-то в сети.
    Для Android-разработчиков и разработчиц главной IDE выступает Android Studio.

Классы

    Для создания класса пишем class и далее — название.
    Можно не писать фигурные скобки. Зачем, если класс пустой.
    У классов есть конструкторы, пустые по умолчанию.
    В Kotlin решили, что писать название класса для объявления конструктора ещё раз — лишняя работа.
    В Kotlin слово constructor опционально, и его можно не писать.
    Слово constructor можно использовать уже внутри класса, когда нам нужно несколько конструкторов. Такие конструкторы называются вторичными (англ. secondary).
    В Kotlin основной конструктор, который указывается при объявлении класса, не может содержать внутри себя код.
    Единственный способ выполнить код при создании класса с первичным конструктором — добавить блок кода init.
    В Kotlin код пишется только во вторичные конструкторы.
    В Kotlin методы можно размещать в теле класса.
    Свойства класса в Kotlin автоматически генерируют get и set методы для каждой переменной.
    В Kotlin можно даже не писать фигурные скобки у методов, а сразу указывать через знак =, какое значение будет возвращать этот метод.
    Дефолтные параметры — значения по умолчанию.
    Именованные параметры — заранее указанные данные в нужном порядке.
    Перечисление, или enum, — это набор именованных констант, который представляет собой ограниченный список идентификаторов.
    В Kotlin четыре модификатора доступа:  private, protected, public, internal.
    private означает видимость только внутри этого класса, включая его члены, как и в Java.
    protected то же самое, что и private, плюс возможность вызывать члены класса в его подклассах.
    В Kotlin модификатор public — модификатор по умолчанию, и если не указывать никакого модификатора, то компилятор подставит именно public модификатор.
    internal позволяет ограничить доступ к файлам в пределах одного модуля.
    Модуль — это набор скомпилированных вместе файлов. В контексте Android-приложения модуль — приложение, а проект (ваше Android-приложение) — контейнер для модулей.
    В Kotlin есть функциональные интерфейсы. Для объявления функционального интерфейса есть специальное ключевое слово fun — как функция.

Наследование и переопределение

    В Kotlin при наследовании и реализации интерфейса используется знак двоеточия :.
    В Kotlin нужно явно указать, что мы хотим наследоваться от данного класса. Для этого используется ключевое слово open.
    В Kotlin есть специальное ключевое слово для переопределения методов — override.
    Ещё одно отличие Kotlin от Java — возможность переопределения свойств.
    В Kotlin тоже есть ключевое слово super, которое позволяет вызывать функции и свойства своего суперкласса.
    Изолированные классы и интерфейсы позволяют выразить ограниченные иерархии классов, которые контролируют наследование.
    Any — это супертип всех non-nullable типов в Kotlin. В том числе для примитивных, таких как Int.
    Поток — сущность, которая помогает выполнять несколько задач параллельно.
    Низкоуровневые компоненты— это компоненты, которые отвечают за самые базовые компоненты систем.
    Тип Unit — это полноценный тип, который может быть использован как аргумент типа.
    Объект класса Unit — это своего рода Singleton, потому что в приложении существует только один экземпляр данного типа, называется он тоже Unit и возвращается неявно.
    Nothing — наследник всех классов в Kotlin, даже отмеченных модификатором final. Но создать его экземпляр не получится, потому что у него есть приватный конструктор.
    Android Studio умеет генерировать простой код за пользователя.

Расширенные возможности классов

    В рабочей практике классы можно разделить на три типа: классы, описывающие какую-то логику, классы с данными (англ. Data Classes) и утилитарные классы.
    Классы с логикой — «рабочая лошадка». На учёбе, а затем на работе вы будете использовать их везде и постоянно.
    Классы с данными хранят в себе только состояние какого-то объекта: их методы обычно не несут логики, а лишь помогают манипулировать состоянием такого класса.
    Функция copy() используется для копирования объекта, что позволяет изменить только некоторые его свойства, оставив остальные неизменными.
    Утилитарные классы обычно не имеют в себе никакого состояния. Более того, часто они даже не решают какую-то единственную проблему, а, скорее, содержат несколько простых решений для часто решаемых проблем.
    Singleton — это паттерн проектирования, который гарантирует, что у класса есть только один экземпляр, и предоставляет к нему глобальную точку доступа.
    Singleton нарушает принцип единой ответственности. В разработке принцип единой ответственности (англ. Single Responsibility Principle) указывает классу на возможность иметь только одну обязанность.
    Объект всегда существует только в одном экземпляре.
    Kotlin позволяет создавать анонимные объекты.
    При создании анонимного класса мы можем создать для него параметры внутри класса.
    В Kotlin анонимные классы могут реализовывать больше одного интерфейса.

Top-level функции

    В Kotlin все функции — функции первого класса.
    Функция высшего порядка (англ. top-level function) — это функция, которая принимает функции как параметры или возвращает функцию в качестве результата.
    Kotlin позволяет расширять класс добавлением новых элементов без необходимости наследования от такого класса с помощью функций-расширений.
    Существуют и свойства-расширения, которые позволяют определять новые свойства для существующих классов.
    Расширения используют те же модификаторы видимости, что и обычные функции, объявленные в той же области видимости.
    Результат остался прежним, но код стал аккуратнее. Такой подход называется деструктурирующим присваиванием.
    Когда мы используем в коде один из операторов, под капотом всегда вызывается соответствующая ему функция.

Дженерики

    В Kotlin нужно задать значение переменной обобщённого типа.
    Если у данного параметра типа есть несколько ограничений, то после угловых скобок нужно написать ключевое слово where, за которым перечислить, от какого класса этот параметр должен наследоваться и какие интерфейсы реализовывать.
    В Kotlin ограничивающие классы и интерфейсы можно перечислять в любом порядке.
    В Kotlin при инициализации экземпляров классов, имеющих дженерик-конструкторы, не нужно указывать тип данных в фигурных скобках, потому что тип данных будет автоматически определён из объекта, который передаётся в конструктор.
    Во множественных ограничениях параметры типа у функции нужно перечислять перед фигурными скобками с использованием ключевого слова where.
    В Kotlin нет возможности или необходимости управлять передачей иерархии наследования. Наоборот, для этого существуют более удобные и гибкие инструменты.
    Чтобы провернуть такой фокус в Kotlin, нам понадобятся два новых ключевых слова: in и out.
    Если объект обобщённого типа передаётся в конструктор, то его тип выводится из самого объекта, и никакие угловые скобки в таком случае не нужны.

Git для самых маленьких

    Git — система контроля версий.
    GitHub — это облачное хранилище, которое позволяет управлять репозиториями Git.
    Проекты в Git представлены репозиториями — хранилищами данных.
    Коммит (англ. commit) — разница между прошлой версией проекта и текущей.
    Форк (англ. fork) — копия репозитория в GitHub.
    Пушить (англ. push) — загружать изменения в удалённый репозиторий.
    Pull Request, или ПР, — это предложение изменения кода в чужом репозитории.
    Ветка (англ. branch) — это история изменений.

Продвинутый Git и Git ignore

    Командная строка — это тоже интерфейс, только текстовый. В ней надо вводить команды, чтобы взаимодействовать с программами.
    Git — это цепочка блоков.
    Блоки в Git — коммиты, каждый из которых имеет ссылку на предыдущий.
    Snapshot («снимок») — снимок данных.
    Этот снимок имеет вид дерева (англ. tree), где ветвями дерева могут быть либо файлы, либо другие деревья (папки). В Git для файлов используется слово blob.
    Ветка — это ссылка на коммит.
    В Git есть специальная ссылка, которая обозначает текущую директорию, и она называется HEAD.
    Хеш коммита — это строка, которая генерируется в зависимости от содержимого коммита. У разных коммитов хеши разные. Сам хеш вычисляется с помощью алгоритма SHA-1.
    В каждом коммите содержатся имя и контакты автора коммита, и они берутся из системных настроек.
    Фаза staging — этап, когда файлы подготовлены, но ещё не содержатся в репозитории.
    Иногда нам не нужно учитывать какие-то изменения. Чаще всего это скрытые файлы или секретные настройки, которые нельзя заливать в публичный репозиторий. Для этого придумали .gitignore файл.
    Чтобы их сохранить — но не в репозиторий, а как будто бы рядом, — есть stash («тайник»).
    Удалённый — не значит находящийся в интернете, а находящийся где-то в другом месте. И неважно, интернет это или соседняя папка.
    Удалённые ветки — это ссылки на коммиты в удалённых репозиториях.
